## BGM 无声 v3 修复：异步时序问题（22:10）

### 问题
用户点击音乐按钮显示"背景音乐开启中..."但始终没声音。根因：`discoverTracks()` 是 async（调用 `wx.cloud.getTempFileURL`），但 `play()` 不等它完成就检查 `TRACKS[trackKey]`——此时 TRACKS 还是空的 `{}`，`play()` 直接 return，啥也不播。

### 修复
1. **`bgm-manager.js` 全量重写 play() 为 async**：
   - `discoverTracks()` 返回 Promise（共享实例，避免重复探测）
   - `play()` `await discoverTracks()` 后再取音轨
   - 找不到 `calm` 时自动回退到 `main` 或第一个可用音轨
   - 新增 `onCanplay` 回调确保音频真正就绪后才播放
   - 返回 boolean 表示播放是否成功
2. **`index.js` toggleBGM() 改为 async**：
   - `await bgm.play('calm')` 等待播放完成
   - 成功显示"道乐悠扬 🎵"，失败显示"暂无可用音轨"

### 修改文件
- `utils/bgm-manager.js` — play() async化 + discoverTracks Promise共享
- `pages/index/index.js` — toggleBGM() async化 + onShow 自动播放修复

## BGM 无声 v4 修复：增加云函数 fallback（22:31）

### 问题
客户端 `wx.cloud.getTempFileURL` 返回15个结果但每个 `tempFileURL` 均为空，导致15首云存储文件无法播放。

### 修复
- `discoverTracks()` 改为两步探测：
  1. 先尝试客户端 `getTempFileURL`
  2. 若返回0首，自动 fallback 调用 `listCloudFiles` 云函数（服务端权限更可靠）
  3. 云函数返回 fileID 后再走一次 `getTempFileURL` 获取临时链接
- 提取 `processFoundTracks()` 公共函数，复用客户端和云函数两条路径

### 修改文件
- `utils/bgm-manager.js` — discoverTracks 增加云函数 fallback + processFoundTracks

### 部署要求
- 需重新上传 `listCloudFiles` 云函数（已支持 candidates 批量探测模式）

## BGM 无声 v5-v6 修复：downloadFile 也失败 → 改用 getFileUrls 云函数（22:45-22:58）

### v5 失败
`wx.cloud.downloadFile` 客户端调用也返回 `internal server error: empty download url`——和 `getTempFileURL` 一样的权限问题。

### v6 根因与修复
- **根因**：客户端调用任何云存储 API（`getTempFileURL`、`downloadFile`）都因为开发环境权限问题返回空的下载链接
- **关键发现**：`cloud-image.js` 通过 **`getFileUrls` 云函数**（服务端 admin 权限）成功获取了87张图片的 HTTPS 临时链接
- **修复**：BGM 照搬 cloud-image.js 模式——调用 `getFileUrls` 云函数获取 HTTPS 临时链接 → 直接传给 `InnerAudioContext`
  - 云函数在服务端用 admin 权限调用 `cloud.getTempFileURL`，绕过了客户端限制
  - `InnerAudioContext` 原生支持 HTTPS URL，无需 downloadFile
  - 缓存获取的 HTTPS 链接（100分钟有效期），避免重复请求
  - 支持 `_prefetchUrls()` 批量预热多首音轨

### 修改文件
- `utils/bgm-manager.js` — 全量重写 v6，使用 getFileUrls 云函数模式

### 部署要求
- 无。`getFileUrls` 云函数已部署（cloud-image.js 在用）

## 冒险分享卡片缺失图片修复（23:15）

### 问题
用户截图反馈：adventure 结果页生成的分享卡片，上方应该显示结果图片（道教人物画像），但显示为空白暗色区域。

### 根因
`generateCard()` 方法里完全没有 `drawImage` 调用，结果对象里的 `imageUrl` 字段（cloud:// 云存储路径）从未被使用。

### 修复
1. 将 `generateCard()` 拆分为三个方法：
   - `generateCard()` — 主入口，获取 canvas 后调用后续步骤
   - `_drawCardBase()` — 画背景、品牌、分隔线（不依赖异步图片）
   - `_drawCardImageThenExport()` — 异步加载图片 → 画图片 → 画文字 → 导出
2. 图片绘制：
   - 使用 `resolveCloudUrl()` 将 `cloud://` 转为 HTTPS 临时链接（cloud-image.js 已有能力）
   - 图片用 `canvas.createImage()` 异步加载，画到圆角裁剪区域内
   - 图片区域：200x200，居中，圆角 12px，位于品牌下方、标题上方
   - 图片失败时继续生成无图片的卡片（不打断）
3. 文字布局调整：所有文字内容下移到图片下方（y=304 起）

### 修改文件
- `pages/adventure/adventure.js` — `generateCard()` 重构 + 新增 `_drawCardBase()` / `_drawCardImageThenExport()` / `_drawImageToCanvas()`
