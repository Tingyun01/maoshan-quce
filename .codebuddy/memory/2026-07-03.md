## 仙山音境：全代码合成音频系统 + 音乐生成器（17:54）

### 核心思路
- 零文件·零版权·纯 Web Audio API 代码合成
- 不引入任何外部音频文件，不增加包体积
- Web Audio 合成的音乐在法律上属于原创作品，无版权风险

### audio-engine.js v3 升级（utils/audio-engine.js）

**保留原有 v2 全部功能**：playClick/Select/Flip/Back/Alert/Enter/Reveal/Complete/Rare

**v3 新增四大音境（背景氛围）**：
| 音境 | 合成方式 | 元素 |
|------|---------|------|
| 🏯 道观晨钟 | 低音嗡鸣+钟声循环+风声 | Drone(65Hz)+BellCycle(12s间隔)+Wind |
| 💧 山间流水 | 流水噪声+鸟鸣+微风 | Bandpass Noise×3层+Birds随机鸣叫+Wind |
| 🎋 竹林风吟 | 风为主+中音嗡鸣+风铃 | Wind强化+Drone(98Hz)+高音Chime随机 |
| 🌧️ 雨夜听松 | 雨声+低嗡+闷雷 | Highpass Noise+Drone(55Hz)+ThunderRumble |

**v3 乐器合成**：
- `playGuqin(noteFreq, velocity)` — 古琴拨弦模拟（Karplus-Strong简化+五层泛音+手指摩擦音）
- `playGuqinPhrase(length, tempo)` — 古琴即兴旋律（五声音阶随机序列）
- 泛音包含微失谐（±8cents），模拟真实琴弦不完美

**v3 自然音合成**：
- 流水：三层带通滤波白噪声+低频LFO水流波动（0.3-0.8Hz）
- 风声：带通滤波噪声+双LFO调制（振幅0.08Hz+滤波频率漂移0.05Hz）
- 鸟鸣：正弦波随机频率(1500-4000Hz)+频率上下滑动+40%概率回应声
- 雨声：高通白噪声+三角波节奏调制
- 闷雷：低通噪声突发(150Hz)+2.5秒衰减，15-40秒随机间隔

**控制API**：
- `startAmbient(theme)` / `stopAmbient()` / `switchAmbient(theme)`
- `previewAmbient(theme)` — 试听4秒
- `setAmbientVolume(0-0.3)` / `getAmbientVolume()`
- `isAmbientActive()` / `getAmbientTheme()`

## 自动跑测系统搭建（18:10）

### 目标
每次更新后自动跑N遍完整用户旅程，模拟从打开→同意协议→首页→答题→结果→图鉴→音境→法务的全流程，发现错误漏洞。

### 技术方案
基于微信官方 `miniprogram-automator`（v0.12.1），连接微信开发者工具CLI实现自动化。

### 文件结构
```
tests/
├── package.json       # 依赖: miniprogram-automator, chalk
├── config.js          # 配置: 用户旅程定义、路径、等待时间
├── flow-full.js       # 执行引擎: 单轮完整旅程跑测
├── cli-runner.js      # CLI入口: 多轮调度、报告生成
└── report.js          # 报告查看器: 查看历史报告
```

### 用户旅程（config.js USER_JOURNEY 数组）
共约45步，分11个阶段：
1. 启动→欢迎遮罩→隐私协议同意→新手引导关闭
2. 首页元素完整性检查（7个关键区域）
3. BGM音乐开关 开/关
4. 全部道法展开/收起
5. 随缘一测→quiz答题5题→提交→结果页
6. 结果页内容检查→返回首页
7. 底部图鉴→统计检查→返回
8. 我的→仙山音境→选音境→古琴即兴→停止→返回
9. 隐私政策页→返回→服务协议页→返回
10. 首页核心入口点击→返回
11. 完成标记

### 使用方式
```bash
cd tests && npm install          # 首次安装
node cli-runner.js               # 默认3轮
node cli-runner.js --rounds=5    # 5轮
node cli-runner.js --rounds=1 --fast  # 快速检查单轮
node report.js                   # 查看最新报告
```

## AI 调用方式升级：CloudBase AI+ 直调混元（22:05）

### 背景
用户已领取小程序成长计划免费Token（CloudBase发布密钥JWT），但之前代码用的是HTTP直调混元API（需要手动配API Key），与成长计划的免费Token不兼容。

### 改造方案
查阅微信官方文档后发现：成长计划的免费Token通过 **`wx.cloud.extend.AI`** 在小程序前端直调，**不需要云函数中转**。

### 代码改动

**pages/result/result.js**:
- `_callRealAI()` 完全重写：不再调用云函数，改用 `wx.cloud.extend.AI.createModel("cloudbase").generateText()`
- 新增 `_buildAIPrompt()` 方法：将在云函数里的prompt构建逻辑移到前端
- 混元模型选 `hunyuan-lite`（省token），可随时切 `hunyuan-pro`
- 响应格式兼容 CloudBase AI 的 `choices[0].message.content` + `usage` 结构

**app.js**:
- `_initAICost()` 重写：不再调云函数ping，改为直接检测 `wx.cloud.extend.AI` 是否可用
- 检测逻辑：`wx.cloud.extend.AI.createModel("cloudbase")` 成功 → AI就绪

### 技术要点
- 基础库 3.16.1 ✅（要求 ≥3.15.1）
- 免费Token自动走 CloudBase 环境鉴权，零配置
- aiAnalyze 云函数保留作为HTTP API降级方案（如需切自配Key）
