## 音乐按钮点击无响应 + 关卡卡片闪动特效（02:33）

### 问题诊断

#### Bug 1：音乐悬浮按钮点击无效
**根因分析**：
1. `movable-view` 的 `bindtap` 在小程序中不可靠，拖拽组件会拦截点击事件
2. `onDrag` 中使用 `e.detail.x`（movable-view 内部坐标）对比 `pageX`（页面绝对坐标），坐标系完全错配 → `_isDragging` 几乎总是 `true` → `onToggle` 被跳过
3. 音频 403 错误后 `_playError` 未追踪 → `isPlaying()` 仍返回 `true` → UI 卡在旋转状态无法关闭

#### Bug 2：关卡卡片视觉不够吸引
用户要求像电商 App 广告卡片那样闪动、多彩、吸引点击。

### 修复内容

| 文件 | 改动 |
|------|------|
| `utils/bgm-manager.js` | 新增 `_playError` 状态跟踪音频加载错误；`isPlaying()` 增加 `!_playError` 条件；`onError` 设置 `_playError=true`，`onPlay` 重置；`_destroyCtx` 重置 `_playError` 和 `_pendingKey` |
| `components/music-float/music-float.wxml` | movable-view 内的 icon 用 `<view class="mfb-inner" catchtap="onInnerTap">` 包裹，确保点击不被拖拽拦截；移除 movable-view 上的 `bindtap`；新增 `catchtouchmove="onTouchMove"` |
| `components/music-float/music-float.js` | 重写触摸体系：`onTouchMove` 用 pageX/pageY 同一坐标系判断位移 >8px；新增 `onInnerTap` 替代原 `onToggle`（仅 tap 触发）；`onDrag` 只保存位置不再混用坐标判断拖拽；`_handleToggle` 先检查真实 `isPlaying()` 状态 |
| `components/music-float/music-float.wxss` | 新增 `.mfb-inner` 样式（填满圆形、居中、:active 缩放反馈） |
| `pages/guide/guide.wxml` | 当前卡片：新增 `.cc-shimmer`（闪光扫过层）+ `.cc-rainbow-border`（彩虹七彩边框）；下一站卡片：新增 `.ns-shimmer`（绿色闪光）；小地图解锁点：新增 `.ms-sparkle`（七彩粒子）+ 锁住点：新增 `.ms-tease`（虚线诱惑闪烁） |
| `pages/guide/guide.wxss` | 新增约 150 行闪卡动画 CSS：`rainbowBorder`（4秒七彩循环）、`shimmerSweep`（3.5秒白色光柱扫过）、`cardPulse`（2.5秒呼吸脉冲）、`sparkleBurst/sparkleDot/sparkleDotAlt`（解锁点七彩粒子旋转闪烁）、`teasePulse`（锁住点虚线呼吸诱惑）、`currentGigaPulse`（当前站点强脉冲金光）、`okGlowCycle`（解锁点彩色光晕轮转） |

### 技术要点
- `movable-view` + `direction="all"` 会吃掉 `bindtap`，解决方法是内部放一个 `catchtap` 的 view 元素拦截点击
- 音频错误状态必须持续追踪（`_playError`），否则 `isPlaying()` 在 ctx 存在但 src 无效时仍返回 true
- 闪卡特效全部纯 CSS 实现，不增加 JS 性能负担，使用 `pointer-events: none` 确保不阻挡点击

## 音乐浮标真机点击无效 + 图鉴抽箴言无响应修复（21:39）

### Bug 1：音乐浮标手机端关不掉
**根因**：v3 版本用 `touchStart/touchMove/touchEnd` 追踪 `_hasMoved` 判断拖拽，真机 movable-view 触摸体系与模拟器不同，`_hasMoved` 状态不可靠导致 `onInnerTap` 被误判跳过。

**修复（music-float v4）**：
- WXML：去掉 catchtap 内层 view 和 touchStart/Move/End 绑定，直接给 movable-view 加 `bindtap="onFloatTap"`
- JS：用 `onDrag` 的 `e.detail.source === 'touch'` 标记真实拖动，`onFloatTap` 检查该标记
- 原理：`bindchange` 只在位置变化时触发，纯点击不触发 → 可以准确区分拖拽和点击
- WXSS：移除已废弃的 `.mfb-inner` 样式

### Bug 2：图鉴详情页点「抽取一枚箴言」无视觉反馈
**根因**：上次 `catchtap=""` → `catchtap="noop"` 的修复未保存，文件被还原。空字符串在微信中不阻止事件冒泡，drawLot() 执行后事件冒泡到 `.detail-overlay` 触发 `closeDetail()`，立即把 `showLotResult` 覆盖为 false。

**症状**：模拟器上有音效（drawLot 执行了）但看不到弹窗；真机上可能完全无反应。
**修复**：album.wxml 100行 `catchtap=""` → `catchtap="noop"`

| 消耗优先级 | 免费→分享 | 免费→分享奖励→时间恢复 |

- `timeRecoveryUsed`：已消耗的时间恢复次数
- 兼容旧版 `count` 字段自动迁移

### 新增工具函数
- `getTimeRecoveryAvailable(limit)`：计算当前可用时间恢复次数
- `getNextRecoverySeconds(limit)`：计算距下次恢复剩余秒数
- `formatRecoveryTime(seconds)`：格式化为"X分XX秒"
- `_getRemainingData(limit)`：统一返回所有剩余数据，供多处复用
